# TDD Process

Описание TDD-цикла для агентов.

## Этапы TDD

```
1. RED    → Тесты написаны, код не написан → тесты FAIL
2. GREEN  → Написан минимальный код → тесты PASS
3. REFACTOR → Код улучшен → тесты всё ещё PASS
```

## Для Tester (пишет тесты)

### До написания кода
1. Изучи архитектурную документацию
2. Определи все сценарии (позитивные, негативные, граничные)
3. Используй моки для зависимостей

### Формат теста
```kotlin
@Test
fun `scenario description - expected behavior`() {
    // Arrange: подготовка
    val mock = mockk<Dependency>()
    every { mock.method() } returns expected
    
    // Act: действие
    val result = systemUnderTest.action(input)
    
    // Assert: проверка
    assertEquals(expected, result)
}
```

### Чеклист
- [ ] Все сценарии покрыты?
- [ ] Позитивные + негативные?
- [ ] Граничные случаи?
- [ ] Моки корректны?
- [ ] Assertions точны?

## Для Developer (пишет код)

### Шаг 1: RED
```bash
./gradlew test
# Все тесты должны FAIL
```

### Шаг 2: GREEN
Напиши МИНИМАЛЬНЫЙ код для прохождения тестов:
- Не оптимизируй
- Не рефактори
- Просто добейся PASS

### Шаг 3: REFACTOR
Улучши код:
- Убери дубликаты
- Улучши именования
- Примени паттерны
- Соблюдай code style

### Важно
- Рефакторинг ТОЛЬКО после GREEN
- Не меняй поведение
- Тесты должны проходить после каждого изменения

## Чеклист готовности

- [ ] Тесты написаны на моках?
- [ ] Тесты FAIL до реализации?
- [ ] Код написан и тесты PASS?
- [ ] Рефакторинг завершён?
- [ ] Тесты всё ещё PASS?
- [ ] Code style соблюдён?
