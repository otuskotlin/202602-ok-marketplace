# REQ-001: Регистрация и аутентификация

## Business Value
Обеспечить безопасный вход на платформу для компаний с верификацией через email и созданием бизнес-профиля. Один пользователь может выступать в роли продавца (создаёт OFFER) или покупателя (создаёт REQUEST).

## User Stories

**US-001: Регистрация новой компании**
* **As a** Пользователь (в роли продавца или покупателя)
* **I want** Зарегистрироваться на платформе
* **So that** Получить доступ к функционалу размещения объявлений и запросов

### Acceptance Criteria (BDD)
* **Given:** Пользователь находится на главной странице
* **When:** Нажимает "Регистрация" и заполняет форму (email, пароль, название компании, ИНН, телефон)
* **Then:** Система создаёт аккаунт, отправляет email с подтверждением, пользователь попадает в личный кабинет

**US-002: Аутентификация пользователя**
* **As a** Зарегистрированный пользователь
* **I want** Войти в систему
* **So that** Получить доступ к личному кабинету

### Acceptance Criteria (BDD)
* **Given:** Зарегистрированный пользователь с подтверждённым email
* **When:** Вводит email и пароль на странице входа
* **Then:** Система проверяет credentials и перенаправляет в личный кабинет

**US-003: Восстановление пароля**
* **As a** Пользователь, забывший пароль
* **I want** Восстановить доступ к аккаунту
* **So that** Снова войти на платформу

### Acceptance Criteria (BDD)
* **Given:** Пользователь нажал "Забыли пароль?"
* **When:** Вводит email для сброса пароля
* **Then:** Система отправляет ссылку для сброса на email, пользователь может установить новый пароль

## User Scenarios (BDD Style)

**Scenario: Регистрация новой компании**
* **Given:** Пользователь находится на главной странице и нажимает кнопку "Регистрация"
* **When:** Пользователь заполняет форму: email, пароль, название компании, ИНН, телефон
* **Then:** Система создаёт аккаунт, отправляет email с подтверждением, пользователь попадает в личный кабинет

**Scenario: Аутентификация существующего пользователя**
* **Given:** Зарегистрированный пользователь с подтверждённым email
* **When:** Пользователь вводит email и пароль на странице входа
* **Then:** Система проверяет credentials и перенаправляет в личный кабинет

**Scenario: Восстановление пароля**
* **Given:** Пользователь забыл пароль и нажал "Забыли пароль?"
* **When:** Пользователь вводит email для сброса пароля
* **Then:** Система отправляет ссылку для сброса на email, пользователь может установить новый пароль

## System Constraints

| Constraint | Value | Notes |
|-----------|-------|-------|
| Latency (login) | < 200 ms | Время отклика при входе |
| Latency (registration) | < 500 ms | Время создания аккаунта |
| Throughput | 5 req/sec | Пиковая нагрузка Year 2 |
| Availability | 99.5% | SLA для B2B платформы |

## Out of Scope
- Интеграция с Госуслугами (ЕСИА)
- OAuth через Google/Yandex
- Двухфакторная аутентификация (2FA)
- SSO для корпоративных клиентов
