# REQ-003: Создание REQUEST (запрос на покупку)

## Business Value
Позволить пользователям в роли покупателя создавать запросы на покупку конкретных компонентов, чтобы продавцы могли предложить свои условия.

## User Stories

**US-007: Создание REQUEST**
* **As a** Пользователь (в роли покупателя)
* **I want** Создать запрос на покупку
* **So that** Получить предложения от продавцов

### Acceptance Criteria (BDD)
* **Given:** Авторизованный пользователь (в роли покупателя) находится в разделе "Запросы"
* **When:** Нажимает "Создать REQUEST", указывает: название, категорию, требуемое количество, желаемую цену, сроки, прикрепляет спецификацию
* **Then:** Система создаёт REQUEST, запускает поиск релевантных OFFER, уведомляет продавцов

**US-008: Получение предложений на REQUEST**
* **As a** Пользователь (в роли покупателя)
* **I want** Просмотреть предложения продавцов
* **So that** Выбрать лучшее предложение

### Acceptance Criteria (BDD)
* **Given:** Пользователь (в роли покупателя) создал REQUEST
* **When:** Продавцы отправляют свои предложения (OFFER в ответ)
* **Then:** Пользователь видит список предложений с ценами и условиями, может сравнить и выбрать

**US-009: Отмена REQUEST**
* **As a** Пользователь (в роли покупателя)
* **I want** Отменить запрос
* **So that** Прекратить получение предложений

### Acceptance Criteria (BDD)
* **Given:** Пользователь (в роли покупателя) хочет отменить запрос
* **When:** Нажимает "Отменить запрос"
* **Then:** REQUEST получает статус "Отменён", уведомления продавцам отправляются

## System Constraints

| Constraint | Value | Notes |
|-----------|-------|-------|
| Latency (create) | < 500 ms | Время создания REQUEST |
| Latency (matching) | < 1000 ms | Алгоритм подбора OFFER |
| Throughput | 5 req/sec | Пиковая нагрузка Year 2 |
| Availability | 99.5% | SLA для B2B платформы |
| Max offers per request | 50 | Лимит предложений |
