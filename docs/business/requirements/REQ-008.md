**Статус:** Roadmap

# REQ-008: Уведомления

## Business Value
Информировать пользователей о важных событиях: новые предложения, сообщения, статус сделки.

## User Stories

**US-022: Уведомление о новом предложении**
* **As a** Пользователь (в роли покупателя)
* **I want** Получать уведомления о новых предложениях
* **So that** Не пропустить интересные предложения

### Acceptance Criteria (BDD)
* **Given:** Пользователь (в роли покупателя) создал REQUEST
* **When:** Пользователь (в роли продавца) отправляет предложение на REQUEST
* **Then:** Пользователь получает уведомление в приложении и на email

**US-023: Уведомление о новом сообщении**
* **As a** Пользователь
* **I want** Получать уведомления о новых сообщениях
* **So that** Не пропустить важные сообщения

### Acceptance Criteria (BDD)
* **Given:** Пользователь участвует в чате
* **When:** Другой участник отправляет сообщение
* **Then:** Пользователь получает уведомление (push/email)

**US-024: Уведомление о статусе сделки**
* **As a** Пользователь (в роли продавца)
* **I way** Получать уведомления об изменении статуса сделки
* **So that** Быть в курсе продвижения сделки

### Acceptance Criteria (BDD)
* **Given:** Идёт процесс заключения сделки
* **When:** Покупатель подтверждает условия сделки
* **Then:** Пользователь (в роли продавца) получает уведомление о подтверждении

## System Constraints

| Constraint | Value | Notes |
|-----------|-------|-------|
| Latency (push) | < 200 ms | Отправка push |
| Latency (email) | < 1000 ms | Отправка email (async) |
| Throughput | 5 req/sec | Пиковая нагрузка Year 2 |
| Availability | 99.5% | SLA для B2B платформы |
| Email delivery | 95% | SLA почтового сервиса |

## Out of Scope
- SMS-уведомления
- Telegram-бот
- WhatsApp-интеграция
- Настраиваемые рассылки
- Маркетинговые пуши
- Уведомления в мессенджерах
