> **Примечание по реализации:** Matching выполняется **on-demand** при просмотре пользователем своих объявлений. При открытии раздела "Мои объявления" система выполняет поиск релевантных совпадений и показывает их прямо в карточке объявления как релевантные матчинги. # REQ-004: Matching (сведение продавца и покупателя)

## Business Value
Автоматически сопоставлять REQUEST покупателя с релевантными OFFER продавцов для ускорения заключения сделки.

## User Stories

**US-010: Автоматический подбор OFFER к REQUEST**
* **As a** Пользователь (в роли покупателя)
* **I want** Получить автоматический подбор предложений
* **So that** Быстро найти релевантные товары

### Acceptance Criteria (BDD)
* **Given:** Пользователь (в роли покупателя) создал REQUEST на "ПЛК Siemens SIMATIC S7-1200"
* **When:** Система анализирует категорию, название, ключевые слова
* **Then:** Система отбирает релевантные OFFER и показывает пользователю первые 10 совпадений

**US-011: Ручной поиск по REQUEST**
* **As a** Пользователь (в роли покупателя)
* **I want** Просмотреть все доступные предложения
* **So that** Расширить выборку

### Acceptance Criteria (BDD)
* **Given:** Пользователь (в роли покупателя) хочет посмотреть все OFFER
* **When:** Нажимает "Показать все предложения"
* **Then:** Система отображает полный список отфильтрованных OFFER с пагинацией

**US-012: Просмотр релевантных матчей в карточке объявления**
* **As a** Пользователь (в роли продавца)
* **I want** Видеть релевантные REQUEST в карточке своего OFFER
* **So that** Быстро найти покупателей для своего товара

### Acceptance Criteria (BDD)
* **Given:** Пользователь (в роли продавца) открыл "Мои объявления"
* **When:** Система выполняет поиск релевантных REQUEST
* **Then:** Результаты отображаются прямо в карточке объявления как релевантные матчи
