# REQ-009: Монетизация (премиум-тарифы)

## Business Value
Обеспечить монетизацию платформы через платные тарифы для пользователей в роли продавца с расширенными функциями.

## User Stories

**US-025: Выбор тарифа**
* **As a** Пользователь (в роли продавца)
* **I want** Приобрести платный тариф
* **So that** Получить расширенные функции для продвижения

### Acceptance Criteria (BDD)
* **Given:** Пользователь (в роли продавца) находится в разделе "Тарифы"
* **When:** Выбирает тариф "Бизнес" и оплачивает
* **Then:** Тариф активируется, открываются расширенные функции

**US-026: Просмотр доступных функций**
* **As a** Пользователь (в роли продавца)
* **I want** Сравнить тарифы
* **So that** Выбрать оптимальный план

### Acceptance Criteria (BDD)
* **Given:** Пользователь на странице тарифов
* **When:** Сравнивает функции тарифов
* **Then:** Система отображает: базовый, бизнес, enterprise с ценами и возможностями

**US-027: Автосписание за продление**
* **As a** Пользователь
* **I want** Автоматическое продление тарифа
* **So that** Не прерывать работу сервиса

### Acceptance Criteria (BDD)
* **Given:** У пользователя активный платный тариф
* **When:** Подходит дата окончания тарифа
* **Then:** Система списывает средства, тариф продлевается

**Scenario: Выбор тарифа**

## System Constraints

| Constraint | Value | Notes |
|-----------|-------|-------|
| Latency (payment) | < 2000 ms | Обработка платежа |
| Latency (tariff check) | < 100 ms | Проверка тарифа |
| Throughput | 5 req/sec | Пиковая нагрузка Year 2 |
| Availability | 99.5% | SLA для B2B платформы |

## Out of Scope
- Гибкая настройка тарифов
- Биллинг для покупателей
- Интеграция с ЭДО
- API для партнёров
- Программа лояльности
- Реферальная система
