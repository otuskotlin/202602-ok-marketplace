# REQ-002: Создание OFFER (объявление о продаже)

## Business Value
Позволить пользователям в роли продавца размещать объявления о продаже промышленных компонентов с полным описанием товара и условий.

## User Stories

**US-004: Создание OFFER**
* **As a** Пользователь (в роли продавца)
* **I want** Разместить объявление о продаже
* **So that** Покупатели могли найти и приобрести мой товар

### Acceptance Criteria (BDD)
* **Given:** Авторизованный пользователь находится в разделе "Мои объявления"
* **When:** Нажимает "Создать OFFER", заполняет: название, категорию, описание, цену, количество, условия поставки, прикрепляет фото
* **Then:** Система сохраняет OFFER, отображает его в каталоге со статусом "Активно"

**US-005: Редактирование OFFER**
* **As a** Пользователь (в роли продавца)
* **I want** Изменить информацию об объявлении
* **So that** Обновить актуальные данные о товаре

### Acceptance Criteria (BDD)
* **Given:** Пользователь имеет активный OFFER
* **When:** Изменяет цену или описание и сохраняет
* **Then:** Система обновляет данные, изменения отображаются в каталоге

**US-006: Деактивация OFFER**
* **As a** Пользователь (в роли продавца)
* **I want** Временно скрыть объявление
* **So that** Приостановить продажу без удаления объявления

### Acceptance Criteria (BDD)
* **Given:** Пользователь хочет временно скрыть объявление
* **When:** Меняет статус на "Неактивно"
* **Then:** OFFER скрывается из каталога, но сохраняется в истории

## System Constraints

| Constraint | Value | Notes |
|-----------|-------|-------|
| Latency (create) | < 500 ms | Время сохранения OFFER |
| Latency (search) | < 300 ms | Поиск в каталоге |
| Throughput | 5 req/sec | Пиковая нагрузка Year 2 |
| Availability | 99.5% | SLA для B2B платформы |
| Max images | 10 | Фото на одно объявление |
| Max file size | 10 MB | Ограничение на файл |
