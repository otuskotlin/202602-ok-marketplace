# REQ-010: Верификация

## Business Value
Повысить доверие покупателей через проверку документов и подтверждение статуса компании.

## User Stories

**US-028: Запрос верификации**
* **As a** Пользователь (в роли продавца)
* **I want** Пройти верификацию
* **So that** Подтвердить надёжность компании для покупателей

### Acceptance Criteria (BDD)
* **Given:** Пользователь (в роли продавца) загрузил документы (ИНН, ОГРН, реквизиты)
* **When:** Нажимает "Пройти верификацию"
* **Then:** Система принимает документы на проверку, отображает статус "На проверке"

**US-029: Успешная верификация**
* **As a** Пользователь (в роли продавца)
* **I want** Получить статус верифицированного продавца
* **So that** Выделиться среди конкурентов и повысить доверие

### Acceptance Criteria (BDD)
* **Given:** Документы прошли проверку
* **When:** Система подтверждает данные
* **Then:** Пользователь получает значок "Верифицирован", отображается в профиле

**US-030: Отклонение верификации**
* **As a** Пользователь (в роли продавца)
* **I want** Получить информацию об отказе
* **So that** Исправить проблемы и повторить попытку

### Acceptance Criteria (BDD)
* **Given:** Документы не прошли проверку
* **When:** Система выявляет несоответствие
* **Then:** Пользователь получает уведомление с причиной отказа

## System Constraints

| Constraint | Value | Notes |
|-----------|-------|-------|
| Latency (submit) | < 500 ms | Загрузка документов |
| Latency (status) | < 200 ms | Проверка статуса |
| Throughput | 5 req/sec | Пиковая нагрузка Year 2 |
| Availability | 99.5% | SLA для B2B платформы |
| Review time | 48 hours | Макс. время проверки |

## Out of Scope
- Автоматическая верификация через ЕГРЮЛ
- Проверка учредителей
- Финансовый аудит
- Лицензии и сертификаты
- Скоринг поставщиков
- Интеграция с СПАРК
