# REQ-007: Коммуникация (чат)

## Business Value
Обеспечить direct-коммуникацию между покупателем и продавцом для обсуждения деталей сделки.

## User Stories

**US-019: Инициирование чата**
* **As a** Пользователь (в роли покупателя)
* **I want** Написать продавцу
* **So that** Обсудить детали сделки

### Acceptance Criteria (BDD)
* **Given:** Пользователь (в роли покупателя) просматривает OFFER продавца
* **When:** Нажимает "Написать продавцу"
* **Then:** Система открывает чат, отправляет уведомление продавцу

**US-020: Обмен сообщениями**
* **As a** Пользователь
* **I want** Общаться в чате в реальном времени
* **So that** Оперативно обсуждать детали сделки

### Acceptance Criteria (BDD)
* **Given:** Открыт чат между двумя пользователями
* **When:** Пользователь отправляет текстовое сообщение
* **Then:** Сообщение сохраняется, отображается у собеседника в реальном времени

**US-021: Прикрепление файла в чате**
* **As a** Пользователь
* **I want** Отправить документ в чате
* **So that** Обменяться спецификациями и реквизитами

### Acceptance Criteria (BDD)
* **Given:** Открыт чат
* **When:** Пользователь прикрепляет файл (спецификация, реквизиты)
* **Then:** Файл загружается, доступен для скачивания обоим участникам

## System Constraints

| Constraint | Value | Notes |
|-----------|-------|-------|
| Latency (send message) | < 200 ms | Отправка сообщения |
| Latency (real-time) | < 500 ms | Доставка через WebSocket |
| Throughput | 5 req/sec | Пиковая нагрузка Year 2 |
| Availability | 99.5% | SLA для B2B платформы |
| Max file size | 25 MB | Ограничение на вложение |
