### Типаж 1: Коммерческий директор завода моторов

#### 1. Демография

| Параметр | Значение |
|----------|----------|
| Возраст | 40-50 лет |
| Опыт в должности | 5-12 лет |
| Образование | Техническое (инженер-электрик) + MBA / управленческие курсы |
| Доход (личный) | 150-300 тыс. ₽/мес. + бонусы (KPI) |
| Компания | Производство электродвигателей, 100-300 сотрудников |
| Выручка компании | 200-500 млн ₽/год |
| Регион | Екатеринбург / Свердловская обл. |

#### 2. JTBD (Jobs to be Done)

**Основная работа:**
> «Выполнить план продаж и обеспечить рост выручки завода, находя новых клиентов в регионе без снижения маржи»

**Формат:**
- Найти 5-8 новых клиентов в месяц
- Увеличить выручку на 20% год к году
- Сохранить маржинальность не ниже 25%

#### 3. Личные боли

| Боль | Глубина |
|------|---------|
| **Закрытый рынок** | Все клиенты знают друг друга, новых нет — «рынок исчерпан» |
| **Посредники** | Забирают 20-30% маржи, но дают объём — отказаться страшно |
| **Зависимость от 3-5 клиентов** | Если один сорвётся — не выполню план, потеряю бонус |
| **Нет инструментов** | Старые методы (холодные звонки, выставки) не работают |
| **Нет видимости** | Мой сайт не ищут, каталог никто не смотрит |

**Цитата (типичная):**
> «Я отвечаю за продажи, но рынок замкнутый — все друг друга знают, новых клиентов найти не могу. Если один крупный клиент сорвётся — я не выполню план.»

#### 4. Как принимает решение

| Параметр | Значение |
|----------|----------|
| Порог автономности | до 500 тыс. ₽ — сам |
| Согласование | > 500 тыс. ₽ — с гендиректором |
| Срок согласования | 1-2 недели |
| Главный критерий | Надёжность контрагента (история, отзывы) |
| Риск | Срыв поставки — репутационный ущерб заводу |

#### 5. Альтернативы (как сейчас решает проблему)

| Метод | Эффективность | Проблема |
|-------|---------------|----------|
| Холодные звонки | 3-5% конверсия | Не знаю, кому звонить — нецелевой контакт |
| Выставки (Империал, Промышленность) | 10-15 контактов / выставка | Дорого, раз в год, не все клиенты посещают |
| Посредники (30%) | Дают объём | Теряю маржу, теряю клиента |
| Сайт / SEO | < 1% обращений | Не ищут — никто не знает, что мы есть |
| «Сарафанное радио» | 20% сделок | Зависимость от 3-5 клиентов |

#### 6. KPI

| Метрика | Текущее | Целевое (платформа) |
|---------|---------|---------------------|
| Новых клиентов / мес. | 1-2 | 5-8 |
| Выручка / мес. | 20-25 млн ₽ | 30+ млн ₽ |
| Конверсия входящих запросов | 10-15% | 30-35% |
| Доля посредников | 30% | 10% |
| Маржинальность | 22% | 28% |

### Типаж 2: Менеджер по развитию дистрибьютора электроники

#### 1. Демография

| Параметр | Значение |
|----------|----------|
| Возраст | 28-38 лет |
| Опыт в должности | 2-6 лет |
| Образование | Высшее (менеджмент / радиоэлектроника) |
| Доход (личный) | 80-150 тыс. ₽/мес. + бонусы |
| Компания | Дистрибьютор электронных компонентов |
| Выручка компании | 500 млн — 3 млрд ₽/год |
| Регион | Москва / Екатеринбург (филиал) |

#### 2. JTBD (Jobs to be Done)

**Основная работа:**
> «Привлечь X новых клиентов в квартал в своём регионе, выполнить план по категории и избавиться от зависшего товара на складе»

**Формат:**
- 10-20 новых клиентов в квартал по Уралу
- Оборот dead stock < 5%
- Обработать 50+ заявок в неделю

#### 3. Личные боли

| Боль | Глубина |
|------|---------|
| **Неизвестность бренда** | Региональные клиенты не знают компанию — «Кто вы? Не слышали» |
| **Холодные звонки = лотерея** | 90% не по адресу, трачу время |
| **Dead stock** | 15-30 млн ₽ зависших оборотных средств |
| **Нет входящих** | Клиент сам не находит — приходится искать |
| **Скорость** | Если КП дольше 2 часов — теряем клиента |

**Цитата (типичная):**
> «Я не знаю, кто в Екатеринбурге ищет контроллеры! Холодные звонки — это лотерея. Мне нужно, чтобы клиент сам нашёл меня, когда ищет именно то, что я продаю.»

#### 4. Как принимает решение

| Параметр | Значение |
|----------|----------|
| Порог автономности | до 200 тыс. ₽ — сам |
| Согласование | > 200 тыс. ₽ — с руководителем направления |
| Срок согласования | 1-2 дня |
| Главный критерий | Скорость ответа + наличие на складе |
| Риск | Потеря клиента из-за скорости |

#### 5. Альтернативы (как сейчас решает проблему)

| Метод | Эффективность | Проблема |
|-------|---------------|----------|
| Холодные звонки | 5% конверсия | Нецелевые, 90% брака |
| email-рассылки | 1-2% открытий | Не работает без входящих |
| Сайт | 1-3 обращения / мес. | Не ищут, не находят |
| Выставки | 20-30 контактов / год | Дорого, раз в год |
| Прямые продажи ( field ) | 5-10 визитов / нед. | Дорого, медленно |

#### 6. KPI

| Метрика | Текущее | Целевое (платформа) |
|---------|---------|---------------------|
| Новых клиентов / мес. | 4-5 | 15-20 |
| Объём dead stock | 15% | 5% |
| Обработанных заявок / нед. | 25 | 60+ |
| Конверсия в сделку | 20% | 40% |
| Выручка категории / мес. | 15 млн ₽ | 25+ млн ₽ |

### Типаж 3: Гендиректор узкоспециализированного поставщика

#### 1. Демография

| Параметр | Значение |
|----------|----------|
| Возраст | 45-60 лет |
| Опыт в должности | 10-25 лет (собственник) |
| Образование | Высшее техническое (инженер) |
| Доход (личный) | 100-200 тыс. ₽/мес. (дивиденды) |
| Компания | Узкоспециализированный поставщик (датчики, ПЛК) |
| Выручка компании | 30-100 млн ₽/год |
| Регион | Екатеринбург / регион |

#### 2. JTBD (Jobs to be Done)

**Основная работа:**
> «Найти клиентов на нишевый товар, который никто не ищет, и выжить в конкуренции с крупными игроками»

**Формат:**
- 40-60 запросов в месяц
- Конверсия в сделку 25-30%
- Рентабельность 15%+

#### 3. Личные боли

| Боль | Глубина |
|------|---------|
| **Специфический товар** | В поиске не вбивают — «никто меня не ищет» |
| **Давление крупняка** | Siemens, Endress+Hauser берут в 3 раза дороже, но бренд решает |
| **Нет visibility** | Сайт — для галочки, нет входящих |
| **Нет ресурсов** | Сам директор = сам продавец = сам инженер |
| **Нет доступа** | Крупные заводы не пускают «маленьких» |

**Цитата (типичная):**
> «Я делаю отличные датчики — немцы берут в 3 раза дороже за то же самое. Но никто меня не ищет! Нужен прямой доступ к запросам от людей, которые точно знают, что им нужно.»

#### 4. Как принимает решение

| Параметр | Значение |
|----------|----------|
| Порог автономности | Всё сам (собственник) |
| Согласование | Нет (единственное лицо) |
| Срок решения | В течение рабочего дня |
| Главный критерий | Техническая компетентность клиента |
| Риск | Объяснять азы продавцам вместо технического разговора |

#### 5. Альтернативы (как сейчас решает проблему)

| Метод | Эффективность | Проблема |
|-------|---------------|----------|
| Холодные звонки | 1-2% конверсия | Не пускают на уровень ЛПР |
| Сайт | < 5 обращений / мес. | Не ищут нишевые товары |
| Выставки | 5-10 контактов / год | Дорого для маленькой компании |
| Тендерные площадки | 1-2 заявки / мес. | Крупные игроки выигрывают ценой |
| Авито / Юла | 1-2 обращения / мес. | Неформат для B2B |

#### 6. KPI

| Метрика | Текущее | Целевое (платформа) |
|---------|---------|---------------------|
| Запросов / мес. | 8-12 | 40-60 |
| Конверсия в сделку | 12% | 30% |
| Рентабельность | 10% | 18% |
| Новых клиентов / мес. | 2-3 | 10-15 |
| Выручка / мес. | 5-8 млн ₽ | 12-15 млн ₽ |

### Типаж 4: Инженер-конструктор

#### 1. Демография

| Параметр | Значение |
|----------|----------|
| Возраст | 25-40 лет |
| Опыт в должности | 3-10 лет |
| Образование | Высшее техническое (инженер-конструктор, электрик) |
| Доход (личный) | 60-120 тыс. ₽/мес. |
| Компания | Приборостроение / машиностроение |
| Выручка компании | 100-300 млн ₽/год |
| Регион | Екатеринбург / Свердловская обл. |

#### 2. JTBD (Jobs to be Done)

**Основная работа:**
> «Быстро найти нужный компонент для проекта, «поставить в чертёж» и не задержать срок проекта»

**Формат:**
- Время поиска компонента < 1 часа
- 100% успешных запросов (есть цена и наличие)
- 0 задержек проекта из-за компонентов

#### 3. Личные боли

| Боль | Глубина |
|------|---------|
| **Поиск = кошмар** | Гуглю — 200 страниц «купить сейчас» с непонятными магазинами |
| **Нет наличия** | Звоню — либо нет в наличии, либо «перезвоните через неделю» |
| **Цена vs бюджет** | Цена в 2 раза выше бюджета, снабжение говорит «дорого, найди дешевле» |
| **Трата времени** | Трачу 2 дня на поиск — проектирование стоит, я выгляжу как тормоз |
| **Нет инструмента** | Нет единого места, где можно вбить артикул и получить цену и наличие |

**Цитата (типичная):**
> «Хочу просто вбить артикул — и получить цену и наличие. Без танцев с бубном. Без звонков 10 поставщикам. Чтобы снабжение видело: вот цена, вот срок — всё, закупаем.»

#### 4. Как принимает решение

| Параметр | Значение |
|----------|----------|
| Решение о компоненте | Сам выбирает (техническое решение) |
| Заявка в снабжение | Передаёт в отдел снабжения |
| Может «продавить» | Если докажет техническую необходимость |
| Главный критерий | Доступность + срок поставки |
| Риск | Задержка проекта — виноват инженер |

#### 5. Альтернативы (как сейчас решает проблему)

| Метод | Эффективность | Проблема |
|-------|---------------|----------|
| Google / Яндекс | 200 страниц результатов | 90% — непонятные магазины, нет наличия |
| Звонки поставщикам | 2-10 звонков / компонент | Не перезванивают, долго, «нет в наличии» |
| Справочники (Чип и Дип) | 1-2 артикула / час | Не все есть, цена розничная |
| Знакомые поставщики | 3-5 проверенных | Ограниченный ассортимент |
| Авито | 1-2 обращения / запрос | Нет гарантии, «купил кота в мешке» |

#### 6. KPI

| Метрика | Текущее | Целевое (платформа) |
|---------|---------|---------------------|
| Время поиска компонента | 4-8 часов | 30 минут |
| Успешных запросов (есть цена и наличие) | 40% | 75% |
| Задержки проектов | 20% случаев | 5% |
| Компонентов в проекте / мес. | 20-50 | 50-100 |
| Стоимость компонентов / проект | 100-500 тыс. ₽ | Растёт |

### Типаж 5: Менеджер по закупкам

#### 1. Демография

| Параметр | Значение |
|----------|----------|
| Возраст | 30-45 лет |
| Опыт в должности | 5-12 лет |
| Образование | Высшее (менеджмент / логистика) |
| Доход (личный) | 70-130 тыс. ₽/мес. + бонусы |
| Компания | Интегратор / производственная компания |
| Выручка компании | 300-800 млн ₽/год |
| Регион | Екатеринбург / Свердловская обл. |

#### 2. JTBD (Jobs to be Done)

**Основная работа:**
> «Обработать все заявки от производства в срок, найти надёжного поставщика по рыночной цене и не сорвать сроки объекта»

**Формат:**
- 50-80 заявок в неделю
- Срок обработки заявки < 2 дней
- Сорванных сроков < 3%

#### 3. Личные боли

| Боль | Глубина |
|------|---------|
| **Объём заявок** | 30-40 заявок в неделю — физически не успеваю |
| **Поставщики не перезванивают** | 3-5 дней жду ответа — объект стоит |
| **Простой бригады** | 50 тыс. ₽ в день — дорого |
| **Нет рейтинга** | Не знаю, кто надёжный — был случай: поставщик пропал |
| **Цена vs бюджет** | Поставщики завышают, нужно согласовывать |

**Цитата (типичная):**
> «Мне нужно, чтобы мне пришло 5 предложений на почту за 2 часа, а не я бегала и искала. Хочу видеть рейтинг и отзывы. Чтобы не звонить 10 поставщикам, а получить готовые варианты.»

#### 4. Как принимает решение

| Параметр | Значение |
|----------|----------|
| Выбор поставщика | Сама, если цена в пределах бюджета |
| Согласование | Если цена > бюджета +15% — с руководителем |
| Главный критерий | Скорость + надёжность, затем цена |
| Оплата | По факту или с отсрочкой |
| Риск | Сорванный монтаж — виновата закупка |

#### 5. Альтернативы (как сейчас решает проблему)

| Метод | Эффективность | Проблема |
|-------|---------------|----------|
| Обзвон 10 поставщиков | 30-60 мин / заявка | Не перезванивают, долго |
| Excel-база поставщиков | 50-100 контактов | Не обновляется, нет рейтинга |
| 1С / ERP | Только внутренние | Нет внешних поставщиков |
| Тендерные площадки | 1-2 дня | Медленно, не все участвуют |
| «Сарафан» | 30% заявок | Зависимость, ограниченный набор |

#### 6. KPI

| Метрика | Текущее | Целевое (платформа) |
|---------|---------|---------------------|
| Время обработки заявки | 3-5 дней | 1-2 дня |
| Обработанных заявок / нед. | 35 | 80+ |
| Сорванных сроков | 15% | 3% |
| Экономия на закупках | - | 10-15% |
| Надёжных поставщиков | 10-15 | 50+ |

**Формат:**
- Время на закупку < 1 часа
- Цена = рыночная (не завышенная)
- «Не найдено» < 5%

#### 3. Личные боли

| Боль | Глубина |

| **Нет доступа** | Меня не воспринимают всерьёз |

| **Гарантия** | Авито — «найдут что угодно», но без гарантии |

**Цитата (типичная):**
> «Хочу покупать как все — по рыночной цене, без унижения. И чтобы не 50 штук, а 1-2. Чтобы не ездить по складам, а сесть и заказать.»

#### 4. Как принимает решение

| Параметр | Значение |
|----------|----------|
| Решение | Сам (часто собственник) |
| Главный критерий | Скорость (нужно «вчера») |
| Готов переплатить | 10-15%, но без лишних телодвижений |
| Оплата | Перевод на карту / расчётный счёт |
| Риск | Сорванный ремонт — потеря контракта |

| Авито | 30% находят | Нет гарантии, риск |
| Сарафан | 20% | Зависимость |
| Знакомый поставщик | 30% | Ограниченный ассортимент |

| Стоимость закупок / мес. | 1-3 млн ₽ | 3-5 млн ₽ |
