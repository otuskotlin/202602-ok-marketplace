## 3. Идеи для MVP (Версия 1.0)

### 3.1 Smart Alerts (AI-уведомления)

| Параметр | Значение |
|----------|----------|
| **REQ ID** | REQ-0XX (создать) |
| **Приоритет** | P1 |
| **Бизнес-ценность** | Retention — причина возвращаться на платформу |
| **Модель** | Бесплатно (Base) + Pro в подписке |

**User Stories:**
- Как пользователь (в роли покупателя), я хочу получать уведомления о новых предложениях по моим запросам
- Как пользователь (в роли продавца), я хочу получать алерты о росте спроса в моей категории
- Как пользователь, я хочу настраивать канал уведомлений (email, push, Telegram)

**Типы алертов для MVP:**
- Новый ответ на мой запрос (Request)
- Изменение статуса сделки
- Новый пользователь (продавец) в категории (Pro)
- Изменение цены (Pro)

### 3.2 AI-Powered Lead Generation

| Параметр | Значение |
|----------|----------|
| **REQ ID** | REQ-0XX (создать) |
| **Приоритет** | P1 |
| **Бизнес-ценность** | Решение network effect trap — привлечение продавцов |
| **Модель** | 5K₽ за "тёплый" лид |

**User Stories:**
- Как пользователь (в роли продавца), я хочу, чтобы платформа находила потенциальных покупателей для моих товаров
- Как пользователь (в роли продавца), я хочу видеть "тёплые" лиды с контактной информацией

**Механика MVP (ручная + полуавтомат):**
1. AI анализирует открытые тендеры, форумы, отраслевые СМИ
2. Менеджер верифицирует лиды
3. Пользователь (в роли продавца) получает "тёплый" контакт

## 5. Почему остальные идеи НЕ для MVP

### Network Effect Trap (Причина отсеивания #1)

Большинство идей (5-14) требуют работающей платформы с продавцами и покупателями. Без этого:
- Нет данных для AI Image Search
- Нет смысла в Closed Community
- Не работает Industrial Data Marketplace

**Решение:** Сфокусироваться на привлечении (Lead Gen) и удержании (Smart Alerts).

### Resource Constraints (Причина отсеивания #2)

| Идея | Требует | Почему не сейчас |
|------|---------|------------------|
| Guaranteed Deals | Страховой фонд, юристы | Риск для раннего старта |
| 1C Integration | 1C-разработчики | Ограниченная база с 1С |
| Government & Rail | Юридическая экспертиза | Сложные госзакупки |
| AI Image Search | ML-инженеры, данные | Не core для MVP |

### Strategic Focus (Причина отсеивания #3)

**MVP должен:**
1. Доказать Product-Market Fit
2. Решить проблему chicken-and-egg
3. Показать Unit Economics

**Не должен:**
1. Строить экосистему (Community, Wiki)
2. Диверсифицировать монетизацию (Data, MaaS)
3. Выходить на новые рынки (Gov, Rail)
