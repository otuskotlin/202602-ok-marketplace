## 1. Обзор

Данный документ содержит прогноз нагрузки для B2B Маркетплейса Промышленных Компонентов на этапе MVP (Екатеринбург).

**Допущения:**
- География MVP: Екатеринбург + Свердловская область
- Целевая ниша: моторы, электронные компоненты, датчики, ПЛК, приводы
- Модель: информационный обмен (без логистики/склада)

## 3. Целевая Аудитория

### 3.1 Рыночный Потенциал: Екатеринбург

**Параметры расчёта:**
- Население Екатеринбурга: ~1.6 млн
- Количество предприятий: ~50,000 (по данным ЕИС)
- Доля промышленных/производственных: ~8% = 4,000 компаний
- Доля с потенциалом цифровизации закупок: ~15% = 600 компаний

### 3.3 Расчёт Транзакций

```
Формула транзакций:
Transactions = DAU × Avg_Sessions × Conv_Rate

Где:
- Avg_Sessions = 2.5 сессий/день (B2B специфика)
- Conv_Rate = 5% (поиск → запрос → сделка)
- Avg_Deal_Size = 150,000 ₽
```

|------------|-------|-------|-------|

## 4. Расчёт RPS

### 4.1 Формула RPS

```
RPS = DAU × Sessions_Per_User × Pages_Per_Session × Page_Weight / Avg_Session_Duration

Где:
- DAU = 10 (Год 1), 28 (Год 2), 75 (Год 3)
- Sessions_Per_User = 2.5
- Pages_Per_Session = 5 (каталог, карточки, фильтры, корзина, профиль)
- Page_Weight = 0.1 req/page (SPA + API calls)
- Avg_Session_Duration = 600 сек (10 минут)
```

### 4.2 Базовый RPS (Без пиков)

| Год | DAU | Sessions | Req/Session | RPS Base |
|-----|-----|----------|-------------|----------|
| 1 | 10 | 25 | 5 | **0.2** |
| 2 | 28 | 70 | 5 | **0.6** |
| 3 | 75 | 188 | 5 | **1.6** |

### 4.3 Пиковый RPS

```
Peak_Coefficient = 3.0 (B2B: заходы в рабочее время 9:00-18:00)
Search_Factor = 1.5 (поиск создаёт больше нагрузки)
```

| Год | RPS Base | × Peak | × Search | **RPS Peak** |
|-----|----------|--------|----------|--------------|
| 1 | 0.2 | ×3.0 | ×1.5 | **1** |
| 2 | 0.6 | ×3.0 | ×1.5 | **3** |
| 3 | 1.6 | ×3.0 | ×1.5 | **7** |
