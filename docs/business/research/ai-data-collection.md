# Исследование: Сбор данных о российском B2B-рынке с использованием AI-инструментов

**Дата:** 2026-03-25  
**Тип:** Технико-рыночное исследование  
**Статус:** Завершено

---

## Цель исследования

Оценить возможность использования AI-инструментов для сбора данных о российском рынке B2B-маркетплейсов в контексте планирования нового маркетплейса промышленных компонентов.

---

## Методология

1. Анализ доступных AI-инструментов веб-скрейпинга (обзор рынка 2026)
2. Сбор открытых данных о российском B2B-рынке
3. Оценка технической реализуемости сбора данных по типам источников
4. Формирование рекомендаций по инструментам и подходам

---

## Результаты

### 1. Доступные AI-инструменты веб-скрейпинга

| Инструмент | Назначение | Ключевая функция | Стоимость |
|------------|------------|------------------|-----------|
| Firecrawl | LLM-workflows, RAG | Markdown/JSON output, open-source | $19/мес |
| Kadoa | Автономный скрейпинг | Self-healing extractors | Custom |
| Browse AI | No-code | Визуальное обучение | $19/мес |
| Diffbot | Knowledge graphs | AI-powered extraction | $299/мес |
| Bright Data | Масштабный сбор | Proxy-инфраструктура | $1/1K записей |
| Apify | Разработчики | Actor marketplace | $29/мес |

**Технологические тренды 2026:**
- Zero-shot vision extraction (VLM) заменяет DOM-based парсинг
- Self-healing scrapers снижают затраты на поддержку до ~0%
- LLM-ready output (Markdown, JSON) становится стандартом

### 3. Техническая реализуемость сбора данных

| Тип данных | Реализуемость | Примечания |
|------------|---------------|------------|
| Публичные цены и каталоги | Высокая | Стандартный скрейпинг |
| Профили компаний на платформах | Высокая | Публичные бизнес-страницы |
| Объёмы транзакций | Средняя | Платформы редко раскрывают GMV по сегментам |
| Финансы компаний-продавцов | Низкая | Не являются публичными |
| Real-time данные о сделках | Низкая | Проприетарные данные платформ |
| Демография пользователей | Средняя | Доступна частичная агрегатная информация |

### 4. Специфические вызовы российского рынка

1. **Анти-бот защита** — Ozon, Wildberries имеют агрессивную детекцию
2. **Языковой барьер** — Русский текст требует NLP-обработки
3. **Динамический контент** — JavaScript-heavy SPA сложнее парсить
4. **Ограниченный API-доступ** — Официальные API ограничены или платны
5. **Актуальность данных** — Рыночные данные меняются быстро

### 5. Возможности AI при сборе данных

**Что AI позволяет делать:**
- Извлечение структурированных данных из неструктурированных источников
- Автоматическая адаптация к изменениям верстки сайтов
- Перевод и понимание русского текста
- Масштабирование на тысячи страниц

**Что AI НЕ позволяет делать:**
- Получать данные, которых не существует в публичном доступе
- Обходить легальные и технические защиты

## Выводы и рекомендации

### Рекомендуемый план действий

**Фаза 1: База (Неделя 1-2)**
1. Использовать вторичные источники:
   - Отчёт Data Insight (платный)
   - Market sizing от Strategy Partners
   - Инвесторские презентации Ozon
2. Настроить мониторинг: TAdviser, Retail.ru, CNews (разделы e-commerce)

**Фаза 2: Пилотный скрейпинг (Неделя 3-4)**
1. Инструмент: Firecrawl (есть бесплатный tier)
2. Цель: Публичные каталоги и цены
3. Объём: 3-5 ключевых категорий на Ozon/Yandex
4. Выход: Структурированный датасет для планирования MVP

**Фаза 3: Масштабирование (Месяц 2)**
1. Добавить Kadoa или Browse AI для автомониторинга
2. Отслеживать листинги конкурентов по категориям
3. Накопить исторический датасет для анализа трендов

### Оценка бюджета

| Уровень | Бюджет | Состав |
|---------|--------|--------|
| Минимальный (DIY) | $0-50/мес | Firecrawl free tier + ручной труд |
| MVP | $100-200/мес | Firecrawl + мониторинг |
| Enterprise | $500+/мес | Kadoa + кастомная разработка |

### Ограничения подхода

- Данные платформ (реальные объёмы транзакций, финансы продавцов) недоступны через скрейпинг
- Платформы могут блокировать автоматизированный сбор
- Ручной аналитический труд (или готовые отчёты) остаётся предпочтительным для первичного исследования
